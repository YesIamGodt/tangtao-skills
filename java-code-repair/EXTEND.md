# Java Code Repair - EXTEND.md

## Overview

This skill provides automatic code repair for Java code based on the report generated by `java-code-spec-scanner`.

## Usage

To use this skill in Claude Code:

```bash
/tangtao-skills:java-code-repair --report /path/to/report.md [--preview] [--config /path/to/config.json]
```

## Arguments

### Required

- **`--report <path>` or `-r <path>`**: Path to the Markdown report generated by `java-code-spec-scanner`.

### Optional

- **`--repo <path>`**: Path to the Git repository. Default: current directory.
- **`--config <path>`**: Path to the configuration file.
- **`--remote <name>`**: Git remote name. Default: `origin`.
- **`--preview`**: Preview mode. Show what changes would be made without actually modifying files.

## Configuration File

Create a `repair-config.json` file to customize the behavior:

```json
{
  "branch_prefix": "ai-repair",
  "git_remote": "origin",
  "commit_prefix": "fix: ",
  "auto_push": true,
  "preview": false
}
```

## Git Branch Naming

Branches are created in the format: `ai-repair-<timestamp>` where timestamp is `YYYYMMDDHHMM`.

## Commit Message Format

```
fix: 自动修复代码规范问题

修复了 <number> 个问题

- 修复 <rule-id>: <count> 处
- 修复 <rule-id>: <count> 处

扫描时间: 2026-03-27 14:30:00
```

## Supported Fix Rules

### Implemented Rules

- **G.EXP.05**: Adds null checks before using variables that might be null
- **G.PRM.07**: Suggests using try-with-resources (preview only)
- **G.LOG.06**: Removes sensitive information from logs
- **G.TYP.02**: Adds division by zero checks
- **G.OTH.03**: Replaces hardcoded addresses with configuration references
- **G.EDV.01**: Suggests using PreparedStatement to prevent SQL injection (preview only)

### Future Enhancements

- G.CON.14: ThreadLocal cleanup
- G.EDV.03: Command injection prevention
- And more...
