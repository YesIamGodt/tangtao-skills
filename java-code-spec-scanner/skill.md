The scanner detects various types of issues and generates detailed reports with actionable information.## When to Use- **Code Reviews**: Use before submitting PRs to ensure compliance- **Project Audit**: Scan entire codebases for existing issues- **Security Checks**: Identify potential security vulnerabilities- **Quality Assurance**: Maintain consistent code quality across projects## Key Features- **20+ Predefined Rules**: Covers null handling, security, IO operations, concurrency, and more- **Comprehensive Reports**: Generates structured reports with file locations, code snippets, and issue details- **Easy to Use**: One command to scan entire repositories- **Extensible**: New rules can be added with minimal effort- **Lightweight**: No external dependencies required## How to Use### Step 1: Activate the SkillWhen you need to scan a Java repository, activate this skill.### Step 2: Run the ScannerThe scanner will automatically traverse the current directory and its subdirectories for Java files.### Step 3: Review the ReportAfter scanning completes, the tool will generate a detailed report in Markdown format.## Supported RulesThe scanner implements the following specifications:### Exception Handling & Concurrency- **G.EXP.05**: 禁止直接使用可能为null的对象，防止出现空指针引用- **G.CON.02**: 在异常条件下，保证释放已持有的锁- **G.CON.04**: 避免使用不正确形式的双重检查锁- **G.CON.14**: 线程池中的线程结束后必须清理自定义的ThreadLocal变量- **P.03**: 使用相同的顺序请求和释放锁来避免死锁### Security- **G.SEC.03**: 加载外部JAR文件时，不要依赖URLClassLoader和java.util.jar提供的默认自动签名检查机制- **G.OTH.01**: 安全场景下必须使用密码学意义上的安全随机数- **G.OTH.03**: 禁止代码中包含公网地址- **G.LOG.05**: 禁止直接使用外部数据记录日志- **G.LOG.06**: 禁止在日志中记录口令、密钥等敏感信息### File IO- **G.FIO.01**: 使用外部数据构造的文件路径前必须进行校验，校验前必须对文件路径进行规范化处理- **G.FIO.02**: 从ZipInputStream中解压文件必须进行安全检查- **G.FIO.05**: 临时文件使用完毕必须及时删除- **P.04**: 在多用户系统中创建文件时指定合适的访问许可### Data Types & Security- **G.TYP.01**: 进行数值运算时，避免整数溢出- **G.TYP.02**: 确保除法运算和模运算中的除数不为0- **G.TYP.11**: 内存中的敏感信息使用完毕后应立即清0- **P.05**: 外部数据使用前必须进行合法性校验### Injection Prevention- **G.EDV.01**: 禁止直接使用外部数据来拼接SQL语句- **G.EDV.02**: 禁止使用外部数据构造格式化字符串- **G.EDV.03**: 禁止向Runtime.exec()方法或java.lang.ProcessBuilder类传递外部数据- **G.EDV.04**: 禁止直接使用外部数据来拼接XML- **G.EDV.08**: 正则表达式应该尽量简单，防止ReDoS攻击- **G.EDV.09**: 禁止直接使用外部数据作为反射操作中的类名/方法名### Resource Handling- **G.PRM.07**: 进行IO类操作时，必须在try-with-resource或finally里关闭资源### Serialization- **G.SER.04**: 不要序列化直接指向系统资源的句柄- **G.SER.06**: 序列化操作要防止敏感信息泄露- **G.SER.08**: 禁止直接将外部数据进行反序列化### Error Handling- **G.ERR.04**: 防止通过异常泄露敏感信息## Report FormatThe scanner generates a Markdown report with:1. **Summary**: Total files scanned, issues found, severity distribution2. **Detailed Issue List**:   - Issue Type: The rule name   - File Location: Path to the problematic file   - Code Snippet: The offending code   - Problem Details: Description of the issue and why it's a violation## Common Mistakes### 1. Null Pointer Exceptions**Problem**: Directly calling methods on objects that may be null**Fix**: Add null checks before accessing object properties or methods**Example**:```java// ❌ Bad - may throw NPEif (object.getValue() > 0) {    // code}// ✅ Good - null check firstif (object != null && object.getValue() > 0) {    // code}```### 2. SQL Injection**Problem**: Directly concatenating user input into SQL queries**Fix**: Use PreparedStatements with parameterized queries**Example**:```java// ❌ Bad - vulnerable to SQL injectionString query = "SELECT * FROM users WHERE name = '" + userName + "'";Statement stmt = connection.createStatement();ResultSet rs = stmt.executeQuery(query);// ✅ Good - use PreparedStatementString query = "SELECT * FROM users WHERE name = ?";PreparedStatement stmt = connection.prepareStatement(query);stmt.setString(1, userName);ResultSet rs = stmt.executeQuery();```### 3. Improper Resource Handling**Problem**: Not closing IO resources properly**Fix**: Use try-with-resources or finally block**Example**:```java// ❌ Bad - resource may not be closed if exception occursFileInputStream fis = new FileInputStream("file.txt");// read file// ✅ Good - try-with-resources automatically closestry (FileInputStream fis = new FileInputStream("file.txt")) {    // read file} catch (IOException e) {    e.printStackTrace();}```## Implementation Details### ArchitectureThe scanner uses a hybrid approach:- **Regular Expressions**: For simple pattern matching- **Abstract Syntax Tree (AST)**: For complex structural analysis (using javalang library)### Rule EngineEach rule is implemented as an independent function with decorator-based registration for easy extensibility.### Performance- **File Traversal**: Multi-threaded for large codebases- **Caching**: Results are cached for repeated scans- **Exclusions**: Support for ignoring specific directories/files via configuration## ConfigurationThe scanner supports a `.java-scanner-config.json` file for custom configuration:```json{  "excludeDirs": ["target", "build", "test"],  "excludePatterns": ["\\.class$"],  "rules": {    "G.EXP.05": true,    "G.LOG.06": true,    "G.EDV.01": false  }}```
